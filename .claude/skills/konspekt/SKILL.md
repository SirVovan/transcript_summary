# Скилл: Обработка транскрипта выступления

## Файлы скилла

- `preprocessor.py` — подсчёт токенов, нарезка транскрипта (ШАГ 0)
- `tov_spec.md` — ToV-спецификация (базовые правила + динамический блок)
- `subagent_phase1.md` — инструкции для субагентов Фазы 1 (черновая сегментация)
- `subagent_phase2.md` — инструкции для субагентов Фазы 2 (написание текста)
- `brigadir.md` — глобальный обзор, верификация, гармонизация, реконструкция, сборка
- `kartograf_profile_*.md` — профили контента (справочник; детальная переработка — отдельный спек)
- `layer2_widget.md` — виджет (Слой 2, конвертация мастер-MD → JSON → HTML)

---

## Пайплайн

```
Транскрипт
    ↓
ШАГ 0: Оценка размера (preprocessor.py info)
    ↓
    ├─ если ≤ 25K токенов ──────────────────────────────────────────→ ЛЁГКИЙ РЕЖИМ
    │                                                                        ↓
    │                                               Claude сам: сегментация → текст → реконструкция → сборка
    │
    └─ если > 25K токенов
                ↓
          Нарезка на чанки по смысловым границам
                ↓
          ФАЗА 1: Параллельные субагенты (черновая сегментация)
                ↓
          БРИГАДИР: Глобальный обзор → чистовая карта
                ↓
          ФАЗА 2: Параллельные субагенты (написание текста)
                ↓
          БРИГАДИР: Гармонизация + переходы + реконструкция + сборка
                ↓
          Мастер-MD → transcripts/
```

## ШАГ 0: Запуск

### Сбор контекста

Выполнить автоматически, без вопросов к пользователю:

1. Проверить `transcripts/` на наличие `*_мастер.md` — для серийного контекста.
   Если найдены — прочитать и сообщить одной строкой.

2. Запустить:
   ```
   python .claude/skills/konspekt/preprocessor.py info [путь к транскрипту]
   ```
   Получить оценку токенов.

Затем задать пользователю **одним сообщением**:

```
1. Имя спикера (если не указано в транскрипте)
2. Профиль контента:
   - [ ] Лекция
   - [ ] Custdev-интервью
   - [ ] Встреча / синк
   - [ ] Конференция / конференс-доклад
   Если не уверен — Claude определит автоматически по транскрипту.
3. Режим работы:
   - Автоматический (без промежуточных проверок)
   - С проверкой (показывать чистовую карту перед Фазой 2; показать мастер-MD в конце)
```

Начинать работу только после получения ответов.

### Выбор режима

| Токенов | Режим |
| --- | --- |
| ≤ 25K | Лёгкий — один чанк, Claude делает всё сам, субагентов нет |
| > 25K | Полный — нарезка на чанки, параллельные субагенты |

---

## Лёгкий режим (≤ 25K токенов)

Claude выполняет весь пайплайн в одном проходе без делегирования.

### Сегментация

Прочитать транскрипт. Разбить на смысловые сегменты, применяя критерии из `brigadir.md`:
- Один сегмент = одна риторическая функция
- Ориентироваться на сигналы смены режима автора
- Каждый сегмент отвечает на один вопрос

Если транскрипт не содержит явных границ — один сегмент.

### Динамический ToV

Прочитать первые ~2–3K токенов транскрипта. Выделить характерные обороты спикера, темп, фирменные фразы. Записать как `[TOV_DYNAMIC]...[/TOV_DYNAMIC]` блок (~150–200 токенов).

### Написание текста

Для каждого сегмента написать блоки `### Карта` и `### Текст`, следуя правилам из `subagent_phase2.md` и `tov_spec.md`.

### Логическая реконструкция и сборка

Следовать инструкциям из `brigadir.md`:
- Написать раздел «Логическая реконструкция» (абзац прозой + таблица; если 1 сегмент — только абзац)
- Добавить переходы между сегментами (если сегментов > 1)
- Собрать мастер-MD и сохранить в `transcripts/`

## Полный режим (> 25K токенов)

### Нарезка на чанки

1. Запустить:
   ```
   python .claude/skills/konspekt/preprocessor.py windows [транскрипт]
   ```
   Получить окна (~2–3K токенов) вокруг потенциальных точек нарезки.

2. Для каждого окна: найти смысловую границу — конец мысли, абзаца.
   - Уверен → зафиксировать таймштамп, продолжить.
   - Не уверен → показать окно пользователю, объяснить неясность, запросить таймштамп.

3. Запустить нарезку:
   ```
   python .claude/skills/konspekt/preprocessor.py split [транскрипт] --at HH:MM:SS,HH:MM:SS,...
   ```
   Получить chunk-файлы (~15–18K токенов каждый, перекрытие ~500 токенов).

### Динамический ToV

Прочитать первые ~2–3K токенов транскрипта. Выделить характерные обороты спикера, темп, фирменные фразы. Записать как `[TOV_DYNAMIC]...[/TOV_DYNAMIC]` блок (~150–200 токенов).

### Фаза 1: Черновая сегментация (параллельно)

Запустить по одному субагенту на каждый чанк, параллельно.

Прочитать `subagent_phase1.md` и передать его содержимое каждому субагенту в начале промпта.

Каждый субагент получает:
- `[CHUNK]` — содержимое своего chunk-файла
- `[SERIAL CONTEXT]` — краткий серийный контекст (если есть мастер-MD предыдущих частей)
- `[PROFILE]` — профиль контента

Результат: черновая карта чанка (список сегментов с метаданными).

### Бригадир: Глобальный обзор

Прочитать `brigadir.md`. Выполнить раздел «Глобальный обзор»:
- Получить черновые карты всех субагентов
- Проверить по трём критериям
- При неопределённости — точечно прочитать фрагмент транскрипта
- Выдать чистовую навигационную карту

Если режим «с проверкой» — показать чистовую карту пользователю, ждать «ок».

### Фаза 2: Написание текста (параллельно)

Запустить по одному субагенту на каждый сегмент из чистовой карты, параллельно.

Прочитать `subagent_phase2.md` и передать его содержимое каждому субагенту в начале промпта.

Каждый субагент получает:
- `[NAV MAP]` — полная чистовая карта
- `[TOV SPEC]` — содержимое `tov_spec.md` + динамический блок ToV
- `[SEGMENT SPEC]` — запись своего сегмента из чистовой карты
- `[CHUNK]` — chunk-файл, содержащий его сегмент

Результат: `### Карта` + `### Текст` для каждого сегмента.

### Бригадир: Сборка

Прочитать `brigadir.md`. Выполнить разделы «Верификация», «Гармонизация», «Переходы», «Логическая реконструкция», «Сборка мастер-MD».

- Как источник для Слоя 2 (форматы вывода — отдельный пайплайн)
- Как серийный контекст для будущих сессий вместо перечитывания транскрипта

## Слой 2: Виджет

Когда пользователь просит создать виджет из мастер-MD — прочитать `layer2_widget.md` и следовать инструкциям.

Мастер-MD должен уже существовать в `transcripts/` до начала Слоя 2.
