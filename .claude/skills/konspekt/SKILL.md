# Скилл: Обработка транскрипта лекции

## Файлы скилла

- `kartograf.md` — ядро картографа (правила карты смыслов)
- `kartograf_profile_lecture.md` — профиль для лекций
- `kartograf_profile_custdev.md` — профиль для custdev-интервью
- `kartograf_profile_meeting.md` — профиль для встреч
- `kartograf_profile_conference.md` — профиль для конференций
- `kartograf_profile_base.md` — базовый профиль
- `konspektolog.md` — правила конспекта
- `konstitutsiya.md` — конституция конспекта
- `widget.md` — правила виджета
- `karta_example.html` — эталон HTML-карты
- `xlsx_template.py` — скрипт генерации XLSX

---

## Архитектура пайплайна

Три шага выполняются строго последовательно. Каждый следующий опирается на данные предыдущего.

```
ШАГ 1: Карта смыслов  →  ШАГ 2: Конспект  →  ШАГ 3: Виджет
        (обязателен)          (опционально)        (опционально)
```

**Контракт между шагами:**

- ШАГ 2 (конспект) имеет два режима:
  - **полный** — текст пишется + сохраняется как DOCX
  - **внутренний** — текст пишется только для передачи в ШАГ 3, DOCX не создаётся

- Если пользователь говорит «пропустить конспект» при наличии виджета в плане — уточнить: пропустить DOCX-файл или пропустить конспект целиком? Если целиком — виджет будет без полного текста в левой колонке, это допустимо только по явному согласию пользователя.

- ШАГ 3 (виджет) **всегда** требует конспектного текста для левой колонки. Источником может быть: DOCX из шага 2, внутренний конспект шага 2, или готовый DOCX, предоставленный пользователем отдельно.

---

## ШАГ 0: Запуск — сбор контекста

Перед началом работы выполнить автоматически, без вопросов к пользователю:

1. Проверить папку `transcripts/` на наличие готовых карт смыслов (`*_карта_смыслов.html` или `*_карта_смыслов.md`) — для определения серийного контекста. Если найдены — прочитать как контекст стиля и сообщить одной строкой.

2. Проверить папку `transcripts/` на наличие сопроводительной презентации к этому уроку (`.pptx`, `.pdf`, `.md`-вытяжка с тем же именем). Если найдена — учесть при работе (точные формулировки промптов, структура шагов, чек-листы).

Затем задать пользователю **одним сообщением**:

```
1. Имя спикера (если не указано в транскрипте)
2. Какие шаги выполнять:
   - [ ] Карта смыслов (XLSX + HTML)  ← обязательна, если нужны шаги 2 или 3
   - [ ] Конспект (DOCX)
   - [ ] Виджет (HTML)
   Если виджет без DOCX — конспект будет написан внутренне и передан в виджет без сохранения файла.
3. Режим работы:
   - Автоматический (без промежуточных проверок)
   - С проверкой (согласовывать сегментацию и промежуточные результаты)
```

Начинать работу только после получения ответов.

## ШАГ 1: Карта смыслов

**Перед началом прочитать:**
- `kartograf.md`
- `kartograf_profile_[тип].md` — профиль определяется после чтения транскрипта
- `xlsx_template.py`
- `karta_example.html`

**Выходные файлы** (сохранять в `transcripts/` рядом с исходником):
- `Карта смыслов — [название].xlsx`
- `Карта смыслов — [название].html`

**Дополнительно** при завершении шага создать:
- `[название]_карта_смыслов.md` — текстовая копия карты для использования в следующих сессиях вместо перечитывания транскрипта (~3–4K токенов против ~16–24K).

**Режим с проверкой:** предложить сегментацию → провести самопроверку → согласовать с пользователем → выполнить.

---

## ШАГ 2: Конспект

**Перед началом прочитать:**
- `konspektolog.md`
- `konstitutsiya.md`

**Режим «полный»** — если пользователь выбрал DOCX:
- Написать конспект от первого лица спикера
- Сохранить как `Конспект — [название].docx` в `transcripts/`
- Передать текст в ШАГ 3 (если он запланирован)

**Режим «внутренний»** — если DOCX не нужен, но запланирован виджет:
- Написать полный конспект от первого лица спикера
- Не сохранять файл
- Передать текст в ШАГ 3

**Режим «пропущен»** — только по явному согласию пользователя, когда виджет не запланирован или пользователь осознанно принял, что левая колонка виджета будет без полного текста.

---

## ШАГ 3: Виджет

**Перед началом прочитать:**
- `widget.md`

**Источник текста для левой колонки** (в порядке приоритета):
1. Текст из ШАГ 2 текущей сессии
2. Готовый DOCX-конспект, предоставленный пользователем
3. Карта смыслов `.md` — только если пользователь явно принял отсутствие полного конспекта

**Критическое правило:** левая колонка виджета — всегда связный текст от первого лица спикера. Буллеты из карты смыслов — недопустимы.

**Выходной файл:**
- `Виджет — [название].html` в `transcripts/`

**Именование:** `Виджет — [Название лекции].html`

---

## Переработка существующих файлов

Если пользователь просит доработать уже созданный виджет или конспект — читать:
- `[название]_карта_смыслов.md` вместо транскрипта
- Существующий файл виджета/конспекта

Транскрипт перечитывать не нужно.
