## Контекст

Скилл `/konspekt` производит мастер-MD (Слой 1). Слой 2 конвертирует мастер-MD в интерактивный HTML-виджет для отображения конспекта урока в двухколоночном формате.

**Ключевой принцип Слоя 2:** механическая конвертация формата, не редакция. Весь контент из мастер-MD переносится без потерь и без выборки. Левая колонка = полный `### Текст`, правая колонка = полный `### Карта`.

---

## Источник данных

Мастер-MD содержит для каждого сегмента:

```markdown
## Сегмент N | HH:MM:SS–HH:MM:SS | [Тема]

**Тип:** [риторическая роль]
**Спикер:** [имя]

**Ключевая мысль:** [1–2 предложения]

### Карта
- **Термин/тезис:** пояснение
- ...

### Текст
Абзацы прозой...

> **Лайфхак:** текст

> **Важно:** текст

> **Демонстрация:** текст
```

---

## Логическая реконструкция в виджете

Раздел `## Логическая реконструкция` из мастер-MD отображается как вводный блок виджета — до первого сегмента.

### Расположение

Рендерится после заголовка виджета (badge + title) и до навигационных табов сегментов.

### Содержимое

- **Абзац прозой** — полностью, без сокращений.
- **Таблица** — если сегментов > 1 (как в мастер-MD: при одном сегменте таблица не нужна).

### HTML-шаблон

```html
<div class="reconstruction">
  <div class="reconstruction-title">Логическая реконструкция</div>
  <p class="reconstruction-prose">текст абзаца прозой</p>
  <table class="reconstruction-table">
    <thead>
      <tr>
        <th>Сегмент</th>
        <th>Риторическая роль</th>
        <th>Ключевой ход автора</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>1</td>
        <td>роль</td>
        <td>ход</td>
      </tr>
    </tbody>
  </table>
</div>
```

### В JSON

Поле `reconstruction` на верхнем уровне (рядом с `meta` и `segments`):

```json
"reconstruction": {
  "prose": "Полный абзац прозой из мастер-MD",
  "table": [
    {"segment": "1", "role": "Риторическая роль", "move": "Ключевой ход автора"}
  ]
}
```

Если сегмент один — `table` пустой массив `[]`, блок рендерится только с прозой.

## Структура виджета

### JSON-формат (входной файл для `widget_generator.py`)

```json
{
  "meta": {
    "badge": "Тема · Подтема",
    "title": "Название урока · Спикер",
    "out": "Виджет — Название.html"
  },
  "reconstruction": {
    "prose": "Абзац прозой из раздела ## Логическая реконструкция мастер-MD",
    "table": [
      {"segment": "1", "role": "Риторическая роль", "move": "Ключевой ход автора"},
      {"segment": "2", "role": "...", "move": "..."}
    ]
  },
  "prompts": {},
  "segments": [
    {
      "id": "01",
      "type": "concept|method|demo",
      "title": "Тема сегмента",
      "timing": "HH:MM–HH:MM",
      "body": "HTML-контент левой колонки",
      "right": "HTML-контент правой колонки"
    }
  ]
}
```

### Маппинг `type` из риторической роли

| Риторическая роль (из мастер-MD) | type | Цвет |
|---|---|---|
| открытие, мотивация, введение | `concept` | синий |
| инструктаж, настройка, метод | `method` | зелёный |
| демонстрация, практика, разбор | `demo` | оранжевый |

## Правила конвертации: `body` (левая колонка)

`body` = полный блок `### Текст` из мастер-MD, преобразованный в HTML.

### Абзацы

Каждый абзац прозы → `<p>текст</p>`

### Жирный текст

`**текст**` → `<strong>текст</strong>`

### Маркированные списки

```markdown
- пункт
- пункт
```
→
```html
<ul style="margin:4px 0 9px 20px;padding:0">
  <li style="margin-bottom:4px">пункт</li>
</ul>
```

### Нумерованные списки

```markdown
1. пункт
2. пункт
```
→
```html
<ol style="margin:4px 0 9px 18px">
  <li style="margin-bottom:4px">пункт</li>
</ol>
```

### Специальные блоки

#### `> **Лайфхак:**`
```html
<div style="background:#ECF2FB;border-left:3px solid #2562B0;border-radius:0 8px 8px 0;padding:8px 12px;margin:6px 0;font-size:12.5px;line-height:1.55">
  <strong>Лайфхак:</strong> текст
</div>
```

#### `> **Важно:**`
```html
<div style="background:#FAF0E4;border-left:3px solid #96580F;border-radius:0 8px 8px 0;padding:8px 12px;margin:6px 0;font-size:12.5px;line-height:1.55">
  <strong>Важно:</strong> текст
</div>
```

#### `> **Демонстрация:**`
```html
<div style="background:#F0F4F8;border-left:3px solid #607D8B;border-radius:0 8px 8px 0;padding:8px 12px;margin:6px 0;font-size:12.5px;line-height:1.55">
  <strong>Демонстрация:</strong> текст
</div>
```

## Правила конвертации: `right` (правая колонка)

`right` = полный блок `### Карта` из мастер-MD, преобразованный в HTML.

### Карточки (insight cards)

Каждый буллет `### Карта` → одна `.insight`-карточка:

```markdown
- **Термин:** пояснение
```
→
```html
<div class="insights">
  <div class="insight"><strong>Термин:</strong> пояснение</div>
  ...
</div>
```

Карточек в `right` ровно столько, сколько буллетов в `### Карта` — без сокращения.

### Дополнительные элементы правой колонки

Некоторые сегменты содержат элементы помимо карточек. Они сохраняются в `right` до или после `.insights`:

**`pr-block`** — блок с копируемым промптом (если в мастер-MD есть блок с промптом):
```html
<div class="pr-block">
  <div class="pr-label">Промпт для копирования</div>
  <pre class="pr-code">текст промпта</pre>
  <button class="pr-copy" onclick="...">Скопировать</button>
</div>
```

**`.cycle`** — циклическая диаграмма шагов (если в мастер-MD есть нумерованный список шагов, специфичный для процесса):
```html
<div class="cycle">
  <div class="cycle-step">
    <div class="step-num">1</div>
    <div class="step-text">текст шага</div>
  </div>
  ...
</div>
```

> **Примечание:** `pr-block` и `.cycle` — исключения. По умолчанию `right` содержит только `.insights`. Решение о добавлении элементов принимается при конвертации конкретного мастер-MD, а не автоматически.

## Технические ограничения

- Никаких `var()` в inline-стилях — только хардкод hex-цветов.
- Все строки в JSON должны быть без переносов строк (`\n` → явный HTML).
- Кавычки внутри HTML-атрибутов → `&quot;`.
- Генерация HTML через `widget_generator.py` — вызов: `python ".claude/skills/konspekt/widget_generator.py" "путь/к/файлу.json"`

---

## Что НЕ входит в Слой 2

- Редакция или сокращение контента — только формат.
- Логическая реконструкция — переносится полностью как вводный блок (см. раздел выше).
- Заголовочный блок мастер-MD (Спикер/Дата/Длительность/Профиль) — в виджет идут только `meta` поля.
