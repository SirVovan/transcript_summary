## Сегмент 2 | 05:44–29:34 | Настройка окружения Claude Code

**Тип:** инструктаж-практика
**Спикер:** Никита Велс

### Карта

- Claude Code в VS Code — рекомендованная среда: удобная работа с файлами прямо в интерфейсе, виден каждый шаг редактирования
- Режим ByPass Permissions: по умолчанию отключён; включается через settings.json или через прямой промт («Внеси в глобальные настройки settings.json режим работы Bypass Permissions по умолчанию»), затем Ctrl+Shift+P → Reload Window
- settings.json содержит: Bypass Permissions, Agent Teams, разрешённые и запрещённые команды — файл публикуется в канале после эфира
- Терминальный Claude Code открывает возможности, недоступные в расширении: Agent Teams, статус-бар в терминале
- Рекомендованные расширения VS Code: Claude Status Bar (показывает лимиты прямо в интерфейсе), Office Viewer (просмотр Excel/CSV/Word внутри VS Code), Claude Notifier (звуковой сигнал о завершении задачи)
- Прокси можно подключить прямо внутри VS Code — VPN тогда не нужен
- Antigravity: в принципе подходит для простых агентов, но не поддерживает субагентов — это критично для Agent Teams
- Все материалы, промты и настройки публикуются в канале сразу после эфира

### Текст

Смотрите, у меня здесь открыт Claude Code в VS Code, и я работаю в режиме ByPass Permissions. По умолчанию этот режим не активен — это важный момент, потому что у многих был вопрос. Почему VS Code? Потому что здесь очень удобно работать с файлами: справа — диалог с клодом, в центре — файл, слева — дерево папок. Вы видите конкретно, что редактирует ваш клод-код, и можете сразу открыть любой файл. В десктопном приложении клода этого нет.

Вот почему я настоятельно рекомендую также установить терминальный Claude Code, а не только расширение. В расширении недоступны Agent Teams и некоторые другие функции, которые нам сегодня понадобятся. Установить терминальный клод на Mac/Linux и на Windows — разные команды, ссылки кинем в канал.

Как включить ByPass Permissions прямо сейчас? Самое простое: вставьте в ваш Claude Code промт «Внеси в глобальные настройки settings.json режим работы Bypass Permissions по умолчанию». Он спросит подтверждение, внесёт изменения. После этого нажимаете Ctrl+Shift+P, выбираете Reload Window — и у вас режим активирован. Вот, в принципе, и всё.

Если хотите сразу получить мои настройки целиком, я публикую в канале файл settings.json. Там прописаны: ByPass Permissions, Agent Teams, список разрешённых команд и запрещённых — это тоже важно, чтобы агент не запускал то, что ему не нужно автоматически. Один раз принимаете разрешение, перезапускаете окно — всё работает.

Расширения, которые я рекомендую установить: Claude Status Bar — он показывает ваши текущие лимиты прямо здесь снизу, перед глазами; Office Viewer — очень удобно для работы с Excel, CSV и вордовскими файлами прямо внутри VS Code, не нужно открывать отдельные программы; Claude Notifier — потому что клод иногда думает 30–40 минут, и нужен звуковой сигнал о том, что он закончил или спрашивает разрешение. Все ссылки кинем в канал после эфира.

По поводу Antigravity — вопрос был в чате. В принципе подходит для простых агентных систем, вообще без проблем. Но у него нет субагентов. А нам сегодня субагенты нужны будут обязательно — именно поэтому работаем в Claude Code.

Итак, с окружением разобрались. Теперь возвращаемся к нашей агентной системе — точнее, к тому, с чего она начинается: с описания того, чего мы вообще хотим.

## Сегмент 3 | 29:34–01:03:24 | Написание инструкций для агентной системы «AICретарь»

**Тип:** демонстрация + практика
**Спикер:** Никита Велс

### Карта

- Стартовый файл — instructions.md (или любое имя): неформальный текст своими словами — что хочешь получить от каждого агента; нейросетью писать нельзя — «мусор на входе, мусор на выходе»
- Система «AICретарь» — личный исполнительный ассистент из пяти агентов: координатор (= CLAUDE.md), юрист, финансовый, ресёрч-агент, ассистент-календарь
- CLAUDE.md совмещает роль главного координатора: читается при каждом диалоге, описывает архитектуру, агентов, правила запуска
- Юрист: глубокая аналитика юридических документов, проверка на подводные камни и уязвимости, создание новых документов (PDF, Word, Google Docs через MCP)
- Финансовый агент: получает на вход сумму/скриншот из банка/счёт-фактуру, вносит в Google Sheets (через MCP); шаблон таблицы — три листа: дашборд с графиками, лист расходов, лист доходов; если тип платежа неясен — уточняет у пользователя
- Ресёрч-агент: подключается к MCP Firecrawl + Brave Search, собирает информацию по теме из интернета, формирует отчёт в Google Docs
- Ассистент-календарь: подключается к Google Calendar через MCP; задачи — получить список встреч на день/неделю, вносить события, проверять наличие времени, предлагать оптимизацию расписания
- Принцип детализации: не слишком мало, не слишком много — перегруженный контекст даёт плохой результат; первая итерация — верхнеуровневая, потом доработка точечно по каждому агенту
- Разница между скилами и агентами: скилы подгружаются по совпадению запроса, агенты запускаются по чётким условиям из координатора; при 5–6 агентах разница небольшая, при 20–30 — принципиальная
- После написания instructions.md — следующий шаг: создать папку с markdown-файлами для каждого субагента и подключить их к CLAUDE.md

### Текст

Смотрите, у нас есть Markdown-файл под названием instructions.md. По сути, это не инструкция в техническом смысле — это ваши хотелки, продуктовые требования, написанные своими словами. Очень важный момент: не пробуйте писать это через нейросеть. Это очень важно. Если вы это всё напишете через нейросеть, вы получите просто неработающую агентную систему либо систему, которая будет делать ерунду. Нужны именно ваши мысли, ваше видение того, что вы хотите получить.

Систему, которую мы собираем сегодня, я назвал «AICретарь» — личный исполнительный ассистент. Делает руками за меня всю рутину: документы, учёт денег, поиск информации, напоминания. Состоит из пяти агентов.

Первый — координатор. Его задача — понимать, к какому агенту относится каждый запрос. И этот координатор у нас совмещается с CLAUDE.md — главным файлом, который клод читает при каждом новом диалоге. Там описаны все агенты, кто для чего нужен, в каких случаях кого запускать, архитектура проекта, правила запуска. Это наш главный файл.

Второй — агент-юрист. Это должен быть профессиональный юрист РФ с пятнадцатилетним стажем, специализирующийся на документах. Его задачи: глубокая аналитика документов, которые я загружаю (любого формата и расширения); поиск правовых норм РФ и сверка с ними; проверка на подводные камни, уязвимости и несоответствия законодательству; итоговый отчёт с положительными, отрицательными сторонами и рисками. И второй режим — создание документов: если ранее загруженный документ был проанализирован, агент создаёт исправленную копию. Если документа изначально нет — задаёт список уточняющих вопросов, после чего пишет документ с учётом всех законодательных норм. Форматы: PDF и Word через скилы, Google Docs через MCP.

Третий — финансовый агент. Получает на вход либо сумму с описанием, либо скриншот из банка, либо счёт-фактуру — и вносит в финансовую Google Sheets. Перед этим сам создаёт шаблон таблицы: три листа — дашборд (красиво оформлен, с графиками по категориям, детализация по дням), лист расходов (дата, сумма, категория), лист доходов (дата, сумма). Агент автоматически определяет тип платежа — доход или расход. Если не уверен, или не понимает категорию — уточняет у пользователя следующим сообщением. По запросу пользователя подключается к таблице и выдаёт аналитику: расходы за прошлый месяц, доходы и так далее.

Четвёртый — ресёрч-агент. Подключается к MCP Firecrawl — для парсинга сайтов — и Brave Search — для поиска в интернете. Получает тему, анализирует её, собирает информацию из разных источников, формирует отчёт в Google Docs через MCP. В перспективе можно добавить ресёрч по соцсетям.

Пятый — ассистент-календарь. Подключается к Google Calendar через MCP. Четыре основных задачи: получать список встреч и задач на сегодня и на неделю вперёд; по запросу выдавать события на конкретный день; вносить события, проверяя наличие свободного времени; предлагать варианты оптимизации расписания — переносить события, сжимать временные блоки, говорить, когда можно поставить новую задачу.

Вот в чём принцип детализации. Здесь есть две крайности: слишком мало информации и слишком много. Если мы сделаем документ слишком огромный, мы перегрузим контекст агента — и на выходе получим плохо проработанный результат. Поэтому первая итерация — верхнеуровневая, потом мы проходимся по каждому агенту точечно и дорабатываем.

По поводу разницы между скилами и агентами — был вопрос в чате. Скилы нужны тогда, когда агенту не хватает информации для задачи: он подключается к внешнему скилу и получает её. В агентах прописан порядок действий, и главный оркестратор жёстко контролирует, в каком случае запускать каждого субагента. В скилах клод сам решает, коррелирует ли скил с запросом — это немного другая логика. На 5–6 агентах разница небольшая. Но когда агентов 20–30 — разница очень большая.

Итак, главная инструкция нашего агента написана. Следующий шаг — создать папку с markdown-файлами для каждого субагента и раскидать их туда, а потом подключить к CLAUDE.md и запускать в режиме Agent Teams.
